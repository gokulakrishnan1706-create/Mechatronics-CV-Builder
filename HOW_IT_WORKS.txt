========================================================================
             MECHATRONICS CV BUILDER - HOW IT WORKS (ARCHITECTURE & FUNCTIONS)
========================================================================

1. OVERVIEW
This application is a modern, single-page React web application designed specifically to build, analyze, and automatically tailor a Mechatronics Engineering CV based on a given Job Description (JD). It uses a "One Master Shot" architecture, meaning all data flows from a single central state and is rendered instantly on the screen.

------------------------------------------------------------------------

2. CORE TECHNOLOGIES
   - Frontend Framework: React (via Vite for lightning-fast building).
   - Styling: Tailwind CSS (with a custom "Aura Dark" premium aesthetic).
   - Animations: Framer Motion (page transitions, glowing effects, logs).
   - Icons: Lucide React (feather-style crisp SVG icons).
   - AI Engine: Google Gemini 2.5 Flash API (for deep analysis and rewriting).
   - Export Engine: html2pdf.js (for converting the live HTML to A4 PDF).

------------------------------------------------------------------------

3. THE "SINGLE SOURCE OF TRUTH" (DATA FLOW)
   - How it works: At the very top of the app (`App.jsx`), there is a single JSON object holding your entire CV (Name, Experience, Projects, Skills). 
   - State Management: When you edit a text box in the sidebar, or when the AI finishes a synthesis, this central object is updated.
   - Reactivity: Because it's built in React, the moment that central JSON object updates, the A4 Preview on the right instantly re-renders to show the new data without the page reloading.
   - Persistence (Auto-Save): Every time this JSON object changes, `App.jsx` silently saves a copy to your browser's `localStorage` (named 'aura_resume_cache'). If you close the tab and come back, it instantly reloads your last edits.

------------------------------------------------------------------------

4. KEY FUNCTIONS & HOW THEY OPERATE

   A. Manual Editing (The Data Hub Sidebar)
      - Function: Allows you to manually tweak bullet points, names, or dates.
      - How it operates: The `DataHub.jsx` component maps over your JSON data and creates input fields for every single text string. When you type in a box, it fires an `onChange` event, sending the exact path (e.g., "experience.0.bullets.1") back to `App.jsx`, which updates that exact spot in the central JSON object.

   B. The Intelligent Brain (AI Match Engine)
      - Function: To read a Job Description (JD) and completely rewrite your CV to match the requirements, while scoring how well you fit.
      - How it operates:
          1. The user pastes a JD into `MatchEngine.jsx` and hits "Engage".
          2. The app bundles your *current* entire CV JSON and the JD text.
          3. It sends this bundle to Google Gemini via `services/ai.js`.
          4. The AI is instructed (via a hidden "System Prompt") to act as an Executive Copywriter. It analyzes the JD for keywords, calculates a Match Score, identifies Missing Keywords, and rewrites the bullet points and professional summary using maximum technical impact (TF-IDF logic).
          5. The AI returns a brand new JSON object representing the *tailored* CV.
          6. `services/ai.js` strictly parses this JSON. If successful, `App.jsx` completely replaces the old CV object with the beautifully tailored new one.
          7. The UI instantly updates to show the new tailor-made CV.

   C. Live System Console (Terminal UI)
      - Function: To provide real-time, professional feedback while the AI is "thinking".
      - How it operates: As `App.jsx` moves through the stages of calling the AI (Initializing -> Analyzing -> Cross-referencing -> Complete), it pushes log messages into an array called `aiFeed`. The `MatchEngine.jsx` reads this array and renders them line-by-line with animated typewriter and glowing effects, simulating a real hacker terminal.

   D. The Scorecard (Animated Analytics)
      - Function: To boldly display the Match Resonance score computed by the AI.
      - How it operates: `MatchScoreCard.jsx` takes the 0-100% score returned by Gemini. It uses SVG math (circumference and offset) to smoothly animate a glowing ring around the number. It shifts colors depending on the score: Cyan (>80%), Purple (>60%), or Red (<60%).

   E. Revert Protection (The Modal Wipe)
      - Function: If you don't like the AI's changes, you can instantly restore your original "Master" data.
      - How it operates: Clicking "Revert" at the top right triggers a custom React Modal (in `App.jsx`) asking for confirmation. If you confirm, it deletes the `localStorage` cache and replaces the central CV object with the original `resumeData.json` file from your hard drive, instantly reverting the preview.

   F. Exporting (PDF & Word Generation)
      - Function: To download your beautiful live CV into a portable format to send to recruiters.
      - How it operates (PDF): The "Download PDF" button targets the exact HTML `div` that holds the A4 preview. It passes this `div` to a library called `html2pdf.js`. This library essentially takes a super-high-resolution, invisible "screenshot" (using HTML2Canvas) of the CV, scales it up by 3x for crisp text, and drops it precisely onto a digital A4 PDF page before triggering a download.
      - How it operates (Word): The Word export takes the raw HTML structure of your CV, wraps it in hidden Microsoft Office XML tags, converts it into a data URI stream, and downloads it as a `.doc` file so it can be opened and edited in MS Word.
